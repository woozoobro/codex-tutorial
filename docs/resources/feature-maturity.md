# 기능 성숙도 (Feature Maturity)

> Codex 문서·릴리스의 기능 성숙도 레벨 해석법

일부 Codex 기능은 성숙도 라벨 뒤에 ship → 각각의 신뢰성, 변경 가능성, 기대 지원 레벨 이해.

| 성숙도 | 의미 | 가이드 |
| --- | --- | --- |
| **Under development** | 사용 준비 안 됨 | 사용 안 함 |
| **Experimental** | 불안정 + OpenAI가 제거·변경 가능 | 자체 위험으로 사용 |
| **Beta** | 광범위 테스트 ready, 대부분 측면 완성, 일부 측면이 사용자 피드백 기반 변경 가능 | 대부분 평가·파일럿에 OK, 작은 변경 예상 |
| **Stable** | 완전 지원, 문서화, 광범위 사용 ready, 시간 지나며 동작·설정 일관 유지 | 프로덕션 사용 안전, 제거는 보통 deprecation 프로세스 |
